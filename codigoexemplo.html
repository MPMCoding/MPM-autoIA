Perguntas Frequentes Relacionadas

Qual a diferença entre herança e composição em Python?

Herança estabelece uma relação "é um" onde uma subclasse herda características da superclasse. Composição estabelece uma relação "tem um" onde uma classe contém instâncias de outras classes como atributos...

Como funciona a sobrescrita de métodos em Python?

A sobrescrita de métodos permite que uma subclasse forneça uma implementação específica de um método já definido na superclasse. Para acessar o método da superclasse, usa-se super()...

Python suporta herança múltipla?

Sim, Python suporta herança múltipla, permitindo que uma classe herde de várias superclasses. A ordem de resolução de métodos (MRO) determina qual método será chamado em caso de conflito...

Python Fácil

Qual é a diferença entre listas e tuplas em Python?

As listas são mutáveis (podem ser modificadas após a criação) e são definidas com colchetes [ ]. Tuplas são imutáveis (não podem ser alteradas após a criação) e são definidas com parênteses ( ).

SQL Médio

Explique a diferença entre INNER JOIN e LEFT JOIN em SQL.

INNER JOIN retorna registros que têm valores correspondentes em ambas as tabelas. LEFT JOIN retorna todos os registros da tabela à esquerda e os registros correspondentes da tabela à direita (ou NULL se não houver correspondência).

Algoritmos Difícil

Descreva o algoritmo de ordenação QuickSort e sua complexidade de tempo.

QuickSort é um algoritmo de divisão e conquista que seleciona um elemento como pivô e particiona o array ao redor do pivô. Sua complexidade de tempo média é O(n log n), mas no pior caso pode ser O(n²) quando o pivô escolhido é sempre o maior ou menor elemento.

Encapsulamento Avançado em Python

Embora Python não possua modificadores de acesso como private, protected e public como em outras linguagens orientadas a objetos, ele oferece convenções e mecanismos para implementar encapsulamento. Este guia explora técnicas avançadas de encapsulamento em Python.

Convenções de Nome
  • _nome: Atributo/método protegido (convenção) - não deve ser acessado diretamente fora da classe ou subclasses
  • __nome: Atributo/método "privado" - sofre name mangling para dificultar acesso externo
  • __nome__: Métodos especiais/dunder - métodos mágicos do Python
Properties e Descritores

Properties permitem definir métodos especiais que atuam como atributos, possibilitando controle sobre acesso, modificação e deleção: 

class Pessoa:
    def __init__(self, nome):
        self._nome = nome
        
    @property
    def nome(self):
        return self._nome
        
    @nome.setter
    def nome(self, valor):
        if not isinstance(valor, str):
            raise TypeError("Nome deve ser string")
        self._nome = valor

Herança Múltipla e MRO (Method Resolution Order)

Python suporta herança múltipla, permitindo que uma classe herde de várias superclasses. Isso introduz complexidades como o problema do diamante, que é resolvido pelo algoritmo C3 de linearização para determinar a Method Resolution Order (MRO).

Exemplo de MRO
class A:
    def metodo(self):
        return "A"

class B(A):
    def metodo(self):
        return "B"

class C(A):
    def metodo(self):
        return "C"

class D(B, C):
    pass

# A ordem MRO será: D -> B -> C -> A -> object
print(D.__mro__)
# Ao chamar metodo() em uma instância de D, 
# a versão de B será executada devido à MRO

Metaclasses

Metaclasses são "classes de classes" - permitem personalizar a criação e comportamento de classes. Isso possibilita implementar padrões avançados como singletons, registros automáticos, e validação de estrutura de classe.
